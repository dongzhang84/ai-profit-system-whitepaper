# Claude Skill vs 每次重新 prompt · 何时该封装

**日期** · 2026-05-03 (周日) | **支柱** · A · Claude Code

---

## 核心论点

Skill 封装的时机判断：写得早伤灵活性，写得晚浪费重复劳动。我的规则是"三次"。

## 钩子候选

- 什么时候该把 prompt 封装成 Skill？我的规则是"三次"。
- 关于 Skill 我每次都被问一个问题：什么时候写，什么时候不写。

## 主干

- **第一次**：自由 prompt，探索需求
- **第二次**：你会发现自己在复用上次的 prompt 模板，开始有 pattern 感
- **第三次**：立刻开 Skill。不要再推迟

- **但三种场景例外，不该封装**：
  - 一次性探索（封装了之后你也不会再跑第二次）
  - 需要强个人化（Skill 会让它变得过通用，失去个性）
  - 依赖 LLM 本身判断而非流程编排（这种情况 prompt 即可）

## 结论候选

- Skill 不是写给 AI 的。是写给未来三个月的自己的。
