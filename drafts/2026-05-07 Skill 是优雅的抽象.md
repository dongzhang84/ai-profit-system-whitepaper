# Claude Skill 为什么是这几年最优雅的抽象

**日期** · 2026-05-07 (周四★) | **支柱** · A · Claude Code

---

## 核心论点

Skill 不只是 prompt 模板。它是 LLM 时代的"函数"——一个优雅的工程抽象。写过 20 个之后我才意识到这一点。

## 钩子候选

- 写了 20 个 Skill 之后我意识到：这不是 prompt 模板。这是 LLM 版本的 function。
- 软件工程优雅抽象的三条标准：做一件事、做好、可组合。Skill 是 LLM 时代第一个同时满足这三条的。

## 主干

- **传统 prompt = 一次性对话**。没有复用、没有版本、没有组合
- **Skill = 可复用、可组合、可版本化的封装单元**
- **和 Unix philosophy 对照**：do one thing well / compose with pipes / everything is text
- **Skill 的 SKILL.md 就是函数签名**：intent / input / output / side effects 都在这里
- **为什么这个抽象是"刚好"的**：不太大（不是 framework 那种上帝类）不太小（不是单个 prompt 那种一次性）

## 结论候选

- 软件工程的优雅抽象 = 做一件事、做好、可组合。Skill 是 LLM 时代第一个满足三条的。
