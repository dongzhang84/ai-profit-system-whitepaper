# Claude Code 让我放弃的 3 个早期习惯

**日期** · 2026-04-26 (周日) | **支柱** · A · Claude Code

---

## 核心论点

做程序员 10 年养成的 3 个"好习惯"，Claude Code 让我在 6 个月里全扔了。

## 钩子候选

- 做程序员 10 年学会的三个"好习惯"，Claude Code 让我 6 个月内全扔了。
- 最反直觉的不是 Claude 会写什么。是它让我重新判断什么叫"好工程"。

## 主干

- **习惯 1：一次写一个完美函数**。丢掉，因为 Claude 试错更快——写 3 个版本跑 eval 比你想清楚再写快
- **习惯 2：先写测试**。丢掉，因为 Claude 修 bug 时会自动补 regression test，前置写 test 变成浪费
- **习惯 3：premature abstraction**。丢掉，因为 Claude 重构的边际成本掉到接近零，按需 abstraction 比提前设计强

## 结论候选

- AI 不只改你用什么工具，还改你对"什么是好工程"的直觉。
- 这篇的后续（5/17）：其中一条我又捡回来了。
