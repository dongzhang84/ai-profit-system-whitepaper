# Skill 写肥了 / 写瘦了 · 我的平衡点

**日期** · 2026-04-23 (周四★) | **支柱** · A · Claude Code

---

## 核心论点

Skill 过长 LLM 读不过来，过短模型每次回头问你。平衡点在"你能在心里复述的长度"。

## 钩子候选

- 第一版 Skill 写了 800 行。第二版 120 行。效果反而好。
- 判断 Skill 写得好不好，有一个最土的测试：你能不能在头脑里复述一遍？

## 主干

- **过肥的症状**：全是 if/else 规则 / 覆盖所有 edge case / 每次调用 context 耗光
- **过瘦的症状**：模型每次问你 clarify / 输出漂移 / 你要补很多 prompt
- **我的三段式平衡**：intent（一句话）+ constraints（3-5 条硬约束）+ examples（1-2 个完整样例）
- **一个翻车案例**：/write-article 初版 800 行，全是否定规则（"不要 X / 不要 Y"），模型彻底没法自主。第二版 120 行，把否定改成正向示例
- **一个做对的案例**：/traffic-review 60 行，三段都到位

## 结论候选

- Skill 的理想长度是你能在心里复述的长度。
