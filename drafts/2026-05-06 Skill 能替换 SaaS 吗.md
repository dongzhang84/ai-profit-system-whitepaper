## 核心论点

## 钩子候选

- 我用一个 Skill 替换了一个 $20/月的 SaaS 产品。结果有点复杂。
- Skill 能不能替换 SaaS？我测了一个用例。省 20 刀 / 花 3 小时 / 多 1 个维护项。

## 主干

- **那个 SaaS 是什么**：选一个具体例子（流量分析 / diagram 生成 / 简单 data transform 类）
- **Skill 版本怎么写的**：SKILL.md 关键部分 + 依赖
- **第一周上手成本**：比 SaaS 高（你要 debug 一次）
- **第一个月后的差异**：可定制度碾压，但功能 completeness 不如 SaaS
- **两条不能替换 SaaS 的理由**：
  - SaaS 自带团队协作 / 权限 / 共享
  - SaaS 更新不用你维护

## 结论候选

- Skill 能替换 SaaS，但只替换那些你每天在用、每天觉得有点不对的 SaaS。
