## 状态约定

填 `状态` 列时：
- `draft` · 骨架在，还没动手写
- `wip` · 开始写了，未发
- `posted` · 已发，X URL 填到旁边括号里
- `skip` · 因当天有别的事件临时跳过（比如热点需要跳线）

## 写作自检（发前必过）

- [ ] Grep `——`（中文破折号）= 0（附录除外）
- [ ] Grep `不是.*[，,]\s*是|不是.*而是` = 0
- [ ] AI 高频中文词汇扫一遍（至关重要 / 意义重大 / 标志着 / 见证 / 根植于 / 不可或缺 等）
- [ ] 戏剧化隐喻扫一遍（被掏空 / 被击穿 / 被重塑 / 时代在呼唤 等）
- [ ] 段首不用 "首先/其次/再者/此外/最后"
- [ ] 每段 2-4 句，无长句塞多个概念
- [ ] 有至少一处第一人称体感（"我试了/我踩过"）
- [ ] 结尾最后 3 段再单独扫一次（AI 味重灾区）
- [ ] 有名有姓有数字，拒绝模糊归属（"研究表明""业界普遍"）

完整清单见 `.claude/skills/write-article/writing-rules.md`。

## Refresh 贴的处理

05-10、05-12、05-14、05-16、05-17、05-19、05-20 是前一版内容的 refresh，不是完全重写。写的时候：
- 开头明确指向上一版（带 X 链接）
- 新内容 = "3 周后的迭代 / 翻车 / 修正"
- 长度可以比原版短（600-1000 字够了）
- 核心价值：展现"持续迭代"这个工程性，也是个人品牌资产
