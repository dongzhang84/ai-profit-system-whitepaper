# 我每天的 Claude Code workflow · 一个 PhD 做 indie 的节奏

**日期** · 2026-04-21 (周二) | **支柱** · A · Claude Code

---

## 核心论点

Claude Code 不是一个工具，是一个开发节奏。一个做 indie 的 PhD 的一天长什么样，告诉你。

## 钩子候选

- 我每天打开电脑第一件事：不是看邮件，是开 Claude Code 的 resume session。
- 我做 indie 七个月了。Claude Code 改的不是我的产出，是我一天的节奏。

## 主干（3-5 bullet）

- **早上 9 点**：resume 昨天的 session，让 Claude 读一遍 AGENTS.md 和当前分支的 last 3 commits
- **上午**：Claude 写代码，我做架构决策和 review。这一段最像"高阶结对编程"
- **下午**：写 Skill 把一天里重复出现的模式固化下来。本周新 Skill: XXX
- **晚上**：更新 AGENTS.md，把 Claude 今天犯的错记一条。下次它自己不犯
- **睡前**：10 分钟 workflow 复盘——今天哪个 prompt 最省时间、哪个在 waste context

## 结论候选

- Claude Code 给你的不是速度，是一天能多做几件事的可能性。
- 我的效率翻倍的地方不在打字速度，在"决策频率"——一天里我做技术决策的次数从 10+ 提到 30+。
