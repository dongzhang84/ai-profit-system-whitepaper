# 我为什么还没用 Agent SDK · 诚实理由

## 核心论点

Agent SDK 是热词，但对我当前场景 Skill 还 cover 得住。不装懂、不装已用。给你 3 个诚实理由 + 开始学的 trigger。

## 钩子候选

- 大家都在写 Agent SDK 教程。我还没碰。给你 3 个真实理由。
- 承认一件事：Agent SDK 我还没上手。不装。这是我现在的判断。

## 主干

- **理由 1：我的场景 Skill 还够用**。举例 /write-article、/traffic-review 这类单任务封装足够
- **理由 2：多一层抽象 = 多一层调试成本**。Skill 的 SKILL.md 心智负担极低，Agent SDK 带状态机、带 router
- **理由 3：等 Skill 撞墙再学也不晚**。学习成本在需要的时候再付，现在付了就忘

- **什么情况下我会立刻学 Agent SDK**（反面的触发点）
  - 多 agent 协作（两个 agent 需要互相评审）
  - 长 horizon 任务（几十步，Skill 层的线性流程撑不住）
  - 工具动态组合（工具组合在运行时才能决定）

## 结论候选

- 把时间花在自己真懂的层次。不要被热词绑架。
- 这篇的后续（5/20）我会老实来承认：过去 3 周里有没有某一条站不住了。
